# HR Components Migration Guide

## Обзор

Этот документ описывает процесс миграции HR функционала из старых компонентов в новые красивые компоненты с современным дизайном.

## Структура новых компонентов

### Новые страницы HR
- `/hr-dashboard` - Новый HR Dashboard с красивыми карточками
- `/hr-candidates` - Управление кандидатами с современным интерфейсом
- `/hr-vacancies` - Управление вакансиями с улучшенным дизайном

### Новые HR компоненты
Все новые HR компоненты находятся в папке `src/app/components/hr/`:

1. **OpenVacanciesCard.tsx** - Карточка с открытыми вакансиями
   - Отображает прогресс по каждой вакансии
   - Цветовая индикация прогресса
   - Кнопки действий для каждой вакансии

2. **TestsChartCard.tsx** - Карточка с графиком тестов
   - Интерактивный график с Chart.js
   - Статистика по тестам
   - Красивые анимации и hover эффекты

3. **WeakQuestionsCard.tsx** - Карточка со слабыми вопросами
   - Список вопросов с низким средним баллом
   - Цветовая индикация сложности
   - Рекомендации по улучшению

4. **OverdueCandidatesCard.tsx** - Карточка с просроченными кандидатами
   - Список кандидатов с истекшими сроками
   - Индикация срочности
   - Быстрые действия (email, звонок, сообщение)

## Особенности нового дизайна

### Цветовая схема
- **Primary**: #2196f3 (синий) - для основных элементов
- **Success**: #4caf50 (зеленый) - для успешных действий
- **Warning**: #ff9800 (оранжевый) - для предупреждений
- **Error**: #f44336 (красный) - для ошибок и критических элементов

### Иконки
Используются иконки из библиотеки Tabler Icons:
- `IconBriefcase` - для вакансий
- `IconUsers` - для кандидатов
- `IconTrendingUp` - для графиков
- `IconAlertTriangle` - для предупреждений
- `IconClock` - для временных элементов

### Анимации и эффекты
- Hover эффекты на карточках
- Плавные переходы
- Интерактивные элементы
- Современные тени и скругления

## Навигация

Новые HR страницы добавлены в боковое меню в разделе "HR Management":
- HR Dashboard
- Candidates
- Vacancies
- Create Vacancy (расширенное создание)

## API интеграция

Все новые компоненты используют существующий API:
- `apiFetch` утилита для запросов
- Те же эндпоинты, что и в старых компонентах
- Совместимость с существующим бэкендом

## Следующие шаги

### Компоненты для миграции
1. **Templates** - `/hr/templates/` → `/hr-templates/`
2. **Individual Candidate** - `/hr/candidate/[id]/` → `/hr-candidate/[id]/`
3. **Individual Vacancy** - `/hr/vacancy/[id]/` → `/hr-vacancy/[id]/`
4. **Compare Candidates** - `/hr/candidates/compare/` → `/hr-compare/`

### Новые функции
1. **Объединенное создание вакансии и теста** - автоматическая связка
2. **Пошаговый процесс создания** - удобный интерфейс
3. **Встроенный редактор вопросов** - без переключения страниц

### Улучшения дизайна
1. Добавить темную тему
2. Улучшить мобильную версию
3. Добавить анимации загрузки
4. Реализовать drag-and-drop для кандидатов

## Заключение

Новые HR компоненты предоставляют современный, красивый и функциональный интерфейс, сохраняя при этом всю функциональность старых компонентов. Миграция выполнена поэтапно, что позволяет легко переключаться между старыми и новыми версиями.
