## 🌍 Как работает определение языка

### По домену (автоматически)

Middleware в `src/middleware.ts` проверяет домен и устанавливает язык:

- `sofihr.ru` → Русский (ru)
- `sofihr.com` → Английский (en)
- `en.sofihr.ru` → Английский (en)
- `localhost` → Русский (ru) по умолчанию

### Настройка для независимого домена

Для английской версии на отдельном домене просто укажите `.com`:
- Русская версия: `https://sofihr.ru`
- Английская версия: `https://sofihr.com`

Middleware автоматически определит язык и загрузит соответствующие переводы.

## 🚀 Что делать дальше

### Шаг 1: Начните с малого

Выберите один небольшой компонент и оберните тексты в `<Trans>`:

```tsx
// Было
<Button>Создать вакансию</Button>

// Стало
import { Trans } from '@lingui/react';

<Button><Trans>Создать вакансию</Trans></Button>
```

### Шаг 2: Извлеките тексты

```bash
npm run i18n:extract
```

Эта команда найдёт все тексты и добавит их в `.po` файлы.

### Шаг 3: Добавьте переводы

Откройте `src/locales/en/messages.po` и добавьте английские переводы:

```po
msgid "Создать вакансию"
msgstr "Create Vacancy"
```

### Шаг 4: Скомпилируйте

```bash
npm run i18n:compile
```

### Шаг 5: Проверьте

Запустите приложение и проверьте на разных доменах.

## 🔑 Ключевые особенности

### 1. Автоматический сбор текстов

LinguiJS автоматически находит все тексты в проекте:
```bash
npm run i18n:extract
```

### 2. Компиляция в build-time

Переводы компилируются заранее, не замедляют runtime.

### 3. Определение языка по домену

Не нужно вручную переключать - язык определяется автоматически.

### 4. Поддержка TypeScript

Полная типизация из коробки.

### 5. Форматирование дат и чисел

Встроенная поддержка локализации дат, чисел, валют.

## 💡 Рекомендации

### 1. Постепенная миграция

Не нужно переводить всё сразу. Старый и новый подход могут существовать вместе.

### 2. Начните с видимого

Переводите сначала то, что видит пользователь:
- Главная страница
- Навигация
- Основные формы

### 3. Используйте команды

```bash
# После каждого изменения
npm run i18n:extract-compile
```

### 4. Коммитьте .po файлы

Файлы `*.po` должны быть в Git:
```bash
git add src/locales/**/*.po
git commit -m "Add translations for X component"
```

### 5. Не коммитьте .js файлы

Файлы `*.js` в locales генерируются автоматически и добавлены в `.gitignore`.

## 🛠 Troubleshooting

### Проблема: Перевод не работает

**Решение:**
1. Убедитесь, что текст обернут в `<Trans>` или `msg`
2. Запустите `npm run i18n:extract`
3. Добавьте перевод в `.po` файл
4. Запустите `npm run i18n:compile`
5. Перезапустите dev-сервер

### Проблема: Ошибка компиляции

**Решение:**
```bash
# Очистите и пересоберите
rm -rf .next
npm run i18n:compile
npm run dev
```

### Проблема: Не определяется язык

**Решение:**
Проверьте middleware в `src/middleware.ts` и логику в `src/utils/i18n.ts`.
