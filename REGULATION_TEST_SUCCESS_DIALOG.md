# 🎯 Улучшение UX: Диалог после создания теста вместо 4-го шага

## 📝 Что изменилось

### ❌ Было: 4 шага в мастере

```
Шаг 1: Основные настройки
Шаг 2: Выбор регламентов
Шаг 3: Генерация вопросов
Шаг 4: Создание приглашений ← Убрали
```

**Проблемы:**
- ❌ 4-й шаг путал пользователей
- ❌ Не всегда нужно создавать приглашения сразу
- ❌ Нельзя было выбрать "только общую ссылку"
- ❌ Длинный процесс создания

---

### ✅ Стало: 3 шага + диалог выбора

```
Шаг 1: Основные настройки
Шаг 2: Выбор регламентов
Шаг 3: Генерация вопросов

↓ Кнопка "Завершить создание"

┌────────────────────────────────────────┐
│ ✅ Тест успешно создан!                │
│                                        │
│ Что вы хотите сделать дальше?          │
│                                        │
│ [🔗 Сгенерировать общую ссылку]        │
│                                        │
│ [✉️ Отправить приглашения сотрудникам] │
│                                        │
│ [Вернуться к списку тестов]            │
└────────────────────────────────────────┘
```

**Преимущества:**
- ✅ Короче процесс создания (3 шага вместо 4)
- ✅ Понятный выбор после создания
- ✅ Можно выбрать "только общую ссылку"
- ✅ Можно вернуться к списку без создания приглашений

---

## 🎨 Диалог успешного создания

### Варианты действий:

#### 1. 🔗 Сгенерировать общую ссылку
- Создаёт general invitation
- Переходит на страницу приглашений
- Показывает сгенерированную ссылку
- Подходит для массового тестирования

#### 2. ✉️ Отправить приглашения
- Переходит на страницу приглашений
- HR может создать именные приглашения
- Указать конкретные email
- Подходит для выборочного тестирования

#### 3. Вернуться к списку
- Тест создан, можно вернуться
- Приглашения можно создать позже
- Гибкость в workflow

### Новые функции

```typescript
// Показать диалог после создания теста
const handleFinishTestCreation = () => {
  setShowSuccessDialog(true);
};

// Создать общую ссылку и перейти к приглашениям
const handleGenerateGeneralLink = async () => {
  if (!testId) return;

  try {
    const response = await apiFetch(`${API_BASE}/api/regulation-tests/${testId}/invitations`, {
      method: 'POST',
      body: JSON.stringify({
        invitations: [{
          type: 'general',
          expiresInDays: 30,
        }],
      }),
    });

    if (response.ok) {
      router.push(`/hr/regulation-tests/${testId}/invitations`);
    }
  } catch (error) {
    console.error('Error creating general invitation:', error);
    alert('Ошибка при создании общей ссылки');
  }
};

// Перейти к странице приглашений для создания именных
const handleGoToInvitations = () => {
  if (!testId) return;
  router.push(`/hr/regulation-tests/${testId}/invitations`);
};
```

### UI Диалога

```tsx
<Dialog open={showSuccessDialog} onClose={() => setShowSuccessDialog(false)} maxWidth="sm" fullWidth>
```

## 📊 Сравнение workflow

### До (4 шага)

```
1. Основные настройки
2. Выбор регламентов
3. Генерация вопросов
4. Создание приглашений ← обязательно заполнить или пропустить
   ↓
   Кнопка "Создать тест"
   ↓
   → Список тестов
```

### После (3 шага + диалог)

```
1. Основные настройки
2. Выбор регламентов
3. Генерация вопросов
   ↓
   Кнопка "Завершить создание"
   ↓
   ┌─ Диалог ─────────────────┐
   │ Выберите действие:       │
   │ • Общая ссылка           │
   │ • Именные приглашения    │
   │ • Вернуться к списку     │
   └──────────────────────────┘
```

## 💡 Преимущества для пользователя

| Аспект | До | После |
|--------|-----|-------|
| **Шагов в мастере** | 4 | 3 (-25%) |
| **Обязательность приглашений** | Нужно заполнить или пропустить | Опционально после создания |
| **Общая ссылка** | Сложно создать | Одна кнопка |
| **Гибкость** | Низкая | Высокая |
| **Понятность** | 4-й шаг путает | Диалог понятен |

## 🧪 Тестирование

### Сценарий 1: Общая ссылка
1. Создать тест (3 шага)
2. Кликнуть "Завершить создание"
3. В диалоге кликнуть "Сгенерировать общую ссылку"
4. ✅ Должен перейти на страницу приглашений
5. ✅ Должна отобразиться общая ссылка

### Сценарий 2: Именные приглашения
1. Создать тест (3 шага)
2. Кликнуть "Завершить создание"
3. В диалоге кликнуть "Отправить приглашения"
4. ✅ Должен перейти на страницу приглашений
5. ✅ Можно создать именные приглашения

### Сценарий 3: Без приглашений
1. Создать тест (3 шага)
2. Кликнуть "Завершить создание"
3. В диалоге кликнуть "Вернуться к списку"
4. ✅ Должен вернуться к списку тестов
5. ✅ Тест создан без приглашений

### Сценарий 4: Закрыть диалог
1. Создать тест
2. В диалоге кликнуть вне или ESC
3. ✅ Диалог закрывается
4. ✅ Можно открыть снова или выйти
