# Функционал "Переписать/Отправить" в интервью кандидатов

## 📋 Обзор

Добавлен функционал, аналогичный тестам регламентов: после остановки записи ответа кандидат получает возможность **переписать** ответ или **отправить** его на сервер.

Это решает проблему случайных остановок записи, когда кандидаты пугаются и останавливают запись на первом ответе, не имея возможности его исправить.

---

## 🎯 Ключевые изменения

### 1. **Новое состояние `recordedBlob`**

```typescript
const [recordedBlob, setRecordedBlob] = useState<Blob | null>(null);
```

Хранит записанный audio/video blob после остановки записи, но до отправки на сервер.

### 2. **Состояние для анимации удаления**

```typescript
const [deletingMessageIndex, setDeletingMessageIndex] = useState<number | null>(null);
```

Отслеживает индекс реплики, которая сейчас анимируется перед удалением.

### 3. **CSS Анимация fade-out с slide эффектом**

```typescript
const fadeOutSlide = keyframes`
  0% {
    opacity: 1;
    transform: translateX(0) scale(1);
  }
  50% {
    opacity: 0.5;
    transform: translateX(10px) scale(0.95);
  }
  100% {
    opacity: 0;
    transform: translateX(30px) scale(0.8);
  }
`;
```

Реплика плавно исчезает, одновременно сдвигаясь вправо и уменьшаясь в размере.

### 4. **Изменение логики `onstop` в MediaRecorder**

**Было:**
```typescript
mr.onstop = () => {
  const blob = new Blob(chunks, { type: blobType });
  sendBlobAnswer(blob); // Сразу отправка!
  // ...
};
```

**Стало:**
```typescript
mr.onstop = () => {
  const blob = new Blob(chunks, { type: blobType });
  setRecordedBlob(blob); // Сохраняем, но НЕ отправляем
  
  // Обновляем реплику в чате
  if (cameraEnabled) {
    const videoUrl = URL.createObjectURL(blob);
    // Обновляем видео-реплику: "live" → готовое видео
    updateChatMessage(video: videoUrl, text: "🎥 Запись готова. Выберите действие ниже");
  } else {
    // Обновляем аудио-реплику
    updateChatMessage(text: "🎤 Запись готова. Выберите действие ниже");
  }
  // ...
};
```

### 5. **Новые функции**

#### `handleRetake()`
Перезаписывает ответ с анимацией:
```typescript
function handleRetake() {
  setRecordedBlob(null);
  setMediaRecorder(null);
  
  // Находим индекс последней реплики пользователя
  let messageIndexToDelete = -1;
  for (let i = chat.length - 1; i >= 0; i--) {
    if (chat[i].role === 'user' && (chat[i].video || chat[i].text.includes('🎤 Запись'))) {
      messageIndexToDelete = i;
      break;
    }
  }
  
  if (messageIndexToDelete !== -1) {
    // Запускаем анимацию удаления
    setDeletingMessageIndex(messageIndexToDelete);
    
    // Через 400ms удаляем реплику после завершения анимации
    setTimeout(() => {
      setChat((p) => {
        const newChat = [...p];
        for (let i = newChat.length - 1; i >= 0; i--) {
          if (newChat[i].role === 'user' && (newChat[i].video || newChat[i].text.includes('🎤 Запись'))) {
            if (newChat[i].video && newChat[i].video !== "live") {
              URL.revokeObjectURL(newChat[i].video);
            }
            newChat.splice(i, 1);
            break;
          }
        }
        return newChat;
      });
      
      setDeletingMessageIndex(null);
      
      // Запускаем новую запись
      setTimeout(() => {
        startRecording();
      }, 50);
    }, 400); // Длительность анимации
  }
}
```

#### `handleSubmitRecording()`
Отправляет записанный ответ:
```typescript
function handleSubmitRecording() {
  if (recordedBlob) {
    sendBlobAnswer(recordedBlob);
    setRecordedBlob(null);
  }
}
```

### 6. **Обновлённый UI (3 состояния)**

```typescript
{!recording && !recordedBlob ? (
  // Состояние 1: Готов к записи
  <>
    <Button onClick={startRecording}>🎤 Записать ответ</Button>
    <Button onClick={() => setSkipDialogOpen(true)}>⏭️ Пропустить</Button>
  </>
) : recording ? (
  // Состояние 2: Идёт запись
  <Button onClick={stopRecording}>⏹️ Остановить запись</Button>
) : recordedBlob ? (
  // Состояние 3: Запись остановлена, выбор действия
  <>
    <Button onClick={handleRetake}>🔄 Переписать</Button>
    <Button onClick={handleSubmitRecording}>✓ Отправить ответ</Button>
  </>
) : null}
```

### 7. **Применение анимации к реплике в чате**

```typescript
<Box key={i} sx={{
  display: 'flex',
  justifyContent: m.role === 'user' ? 'flex-end' : 'flex-start',
  mb: 1,
  // Добавляем анимацию удаления
  ...(deletingMessageIndex === i && {
    animation: `${fadeOutSlide} 0.4s ease-out forwards`
  })
}}>
```

Когда `deletingMessageIndex` совпадает с индексом реплики, применяется анимация fade-out + slide.

### 8. **Очистка `recordedBlob` при переходе к новому вопросу**

Добавлено в двух местах в `sendBlobAnswer` и `sendEmptyAnswer`:

```typescript
setAnswered(false);
setRecordedBlob(null); // Очищаем для нового вопроса
```

## 🔄 Пользовательский сценарий

### До изменений:
1. Кандидат нажимает "🎤 Записать ответ"
2. Идёт запись (индикатор REC)
3. Кандидат нажимает "⏹️ Стоп"
4. **Ответ сразу отправляется** ❌
5. Невозможно переделать

### После изменений:
1. Кандидат нажимает "🎤 Записать ответ"
2. Идёт запись (индикатор REC)
3. В чате появляется реплика пользователя с видео/аудио
4. Кандидат нажимает "⏹️ Остановить запись"
5. **Появляются 2 кнопки:**
   - 🔄 **Переписать** - удаляет старую реплику из чата и запускает запись заново ✅
   - ✓ **Отправить ответ** - отправляет на сервер ✅
6. Кандидат может переписывать **бесконечное количество раз**
7. **Таймер продолжает идти** (не останавливается при остановке записи)
8. При перезаписи старая реплика **автоматически удаляется** из истории чата

## ⚙️ Технические детали

### Поведение таймера
- Таймер **НЕ останавливается** при остановке записи
- Таймер продолжает отсчёт во время выбора "Переписать/Отправить"
- При истечении времени срабатывает `auto-stop` и отправляется пустой ответ

### Бесконечные переписывания
- Кандидат может нажимать "🔄 Переписать" сколько угодно раз
- Ограничения нет (пока не истёк таймер вопроса)

### Preview записи
- После остановки записи **НЕ показывается** preview видео/аудио
- Это упрощает UI и ускоряет процесс

### Очистка чата при перезаписи
- При нажатии "🔄 Переписать" последняя реплика пользователя удаляется из чата
- **С красивой анимацией:** реплика плавно исчезает с fade-out и slide эффектом (400ms)
- Это работает как для видео (`m.video`), так и для аудио (`m.text.includes('🎤 Запись')`)
- Освобождается память через `URL.revokeObjectURL()` для видео
- Чат остаётся чистым, без "мусора" от предыдущих попыток записи
- Анимация делает UX более плавным и приятным

### Применение
- Функционал применяется ко **всем вопросам** интервью
- Работает одинаково для видео и аудио режима
