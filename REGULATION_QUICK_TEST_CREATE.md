# Быстрое создание теста по регламенту

**Дата:** 03.11.2025  
**Задача:** Добавить возможность создавать тест прямо из списка регламентов одним кликом

---

## 🎯 Проблема

HR приходилось:
1. Открыть список регламентов
2. Запомнить название/ID регламента
3. Перейти в "Тестирование сотрудников"
4. Нажать "Создать тест"
5. Выбрать нужный регламент из списка

**Итого:** 5 шагов + необходимость запоминать название

---

## ✅ Решение

Добавлена кнопка **"Создать тест"** (📋 Assignment icon) прямо в строке каждого регламента.

При клике:
1. ✅ Открывается форма создания теста
2. ✅ Регламент **автоматически выбран**
3. ✅ HR может сразу продолжить настройку

**Итого:** 1 клик! 🚀

   ```typescript

   ```

2. Добавлено чтение URL параметра:
   ```typescript
   const searchParams = useSearchParams();
   const [preselectedFromUrl, setPreselectedFromUrl] = useState(false);
   
   useEffect(() => {
     // Если передан regulationId в URL и регламенты загружены - предвыбираем
     const regulationIdParam = searchParams.get('regulationId');
     if (regulationIdParam && regulations.length > 0 && !preselectedFromUrl) {
       const id = parseInt(regulationIdParam);
       if (!isNaN(id) && regulations.some(r => r.id === id)) {
         setSelectedRegulations([id]);
         setPreselectedFromUrl(true);
       }
     }
   }, [searchParams, regulations, preselectedFromUrl]);
   ```

3. Добавлен визуальный индикатор предвыбора:
   ```tsx

     </Alert>
   )}
   ```

## 🔄 Логика работы

### 1. HR кликает на 📋 в списке регламентов

**URL:** `/hr/regulation-tests/create?regulationId=5`

### 2. Страница создания теста загружается

```typescript
// Шаг 1: Загружаем регламенты
loadRegulations(); // → regulations = [...]

// Шаг 2: Читаем URL параметр
const regulationIdParam = searchParams.get('regulationId'); // → "5"

// Шаг 3: Проверяем что регламент существует
const id = parseInt(regulationIdParam); // → 5
if (regulations.some(r => r.id === id)) {
  // Шаг 4: Предвыбираем регламент
  setSelectedRegulations([id]); // → [5]
  setPreselectedFromUrl(true); // → показываем уведомление
}
```

### 3. HR продолжает создание теста

- ✅ Может оставить только предвыбранный регламент
- ✅ Может добавить дополнительные регламенты
- ✅ Может убрать предвыбранный регламент

## 🧪 Тестирование

### Сценарий 1: Создание теста по одному регламенту

1. Открыть `/hr/regulations`
2. Найти любой регламент
3. Кликнуть на иконку 📋 (Assignment)
4. **Ожидается:**
   - Открылась страница `/hr/regulation-tests/create?regulationId=X`
   - На шаге 2 регламент уже выбран
   - Показано зелёное уведомление "✅ Регламент был автоматически выбран"

### Сценарий 2: Добавление дополнительных регламентов

1. Выполнить Сценарий 1
2. На шаге 2 выбрать дополнительный регламент
3. **Ожидается:**
   - Оба регламента выбраны
   - Количество вопросов = `questionsPerRegulation * 2`

### Сценарий 3: Отмена предвыбранного регламента

1. Выполнить Сценарий 1
2. На шаге 2 снять галочку с предвыбранного регламента
3. **Ожидается:**
   - Регламент убран из выбора
   - Можно выбрать другие регламенты

### Сценарий 4: Создание теста без URL параметра

1. Открыть `/hr/regulation-tests` → "Создать тест"
2. **Ожидается:**
   - Форма работает как раньше
   - Нет предвыбранных регламентов
   - Нет зелёного уведомления

## 🔗 URL структура

**Формат:** `/hr/regulation-tests/create?regulationId={ID}`

**Примеры:**
- `/hr/regulation-tests/create?regulationId=1`
- `/hr/regulation-tests/create?regulationId=42`

**Обработка:**
- ✅ Валидный ID существующего регламента → предвыбор
- ❌ Невалидный ID → игнорируется, показывается обычная форма
- ❌ Несуществующий ID → игнорируется, показывается обычная форма
- ✅ Без параметра → обычная форма
