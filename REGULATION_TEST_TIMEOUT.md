# Обработка истечения времени в тестах по регламентам

## 📋 Описание

Система тестирования по регламентам включает обработку истечения времени на вопрос (`maxTime`). Когда таймер доходит до нуля, выполняется автоматическая отправка ответа на сервер.

---

## ⏱️ Логика auto-submit при истечении времени

### 3 сценария обработки:

#### 1️⃣ **Запись идёт (recording = true)**
```typescript
if (recording && mediaRecorder) {
  // Останавливаем запись
  handleStopRecording();
  
  // После создания blob (через onstop) → отправляем
  handleSubmitAnswer();
}
```

**Поведение:**
- Запись автоматически останавливается
- Создаётся blob с видео/аудио
- Blob отправляется на сервер
- ✅ Ответ засчитан

---

#### 2️⃣ **Запись остановлена, blob готов (audioBlob существует)**
```typescript
else if (audioBlob) {
  // Blob уже есть, но пользователь не нажал "Отправить"
  handleSubmitAnswer();
}
```

**Поведение:**
- Пользователь записал ответ
- Остановил запись
- Не успел нажать "Отправить ответ"
- Система автоматически отправляет готовый blob
- ✅ Ответ засчитан

---

#### 3️⃣ **Запись не начиналась (audioBlob = null)**
```typescript
else {
  // Пользователь вообще не записывал ответ
  handleSubmitEmptyAnswer();
}
```

**Поведение:**
- Пользователь не нажал "🎤 Записать ответ"
- Или открыл вопрос, но ничего не делал
- Отправляется **пустой ответ** на сервер
- ❌ Ответ = 0 баллов

---

## 📤 Отправка пустого ответа

### Frontend (`handleSubmitEmptyAnswer`)

```typescript
const handleSubmitEmptyAnswer = async () => {
  const formData = new FormData();
  formData.append('questionId', currentQuestion.id.toString());
  formData.append('answerText', ''); // ✅ Пустой ответ

  const response = await fetch(
    `${API_BASE}/api/public/regulation-tests/session/${sessionId}/answer`,
    {
      method: 'POST',
      body: formData,
    }
  );

  if (response.ok) {
    skipToNextQuestion(); // Переходим к следующему вопросу
  }
};
```

**Что отправляется:**
- `questionId` - ID вопроса (обязательно)
- `answerText` - пустая строка
- `video` / `audio` - НЕ отправляются

---

### Backend (`RegulationTestPublicController`)

```php
// Контроллер принимает ответ
$answer = new RegulationTestAnswer();
$answer->setSession($session);
$answer->setQuestion($question);
$answer->setAnswerText($data['answerText'] ?? ''); // ✅ Может быть пустым

$this->em->persist($answer);
$this->em->flush();

// Оценка через AI
$this->answerEvaluator->evaluateAnswer($answer);
```

**Обработка в `RegulationAnswerEvaluatorService`:**

```php
public function evaluateAnswer(RegulationTestAnswer $answer): void
{
    // ... транскрипция (если есть видео/аудио)
    
    // Проверка на пустой ответ
    $answerText = trim($answer->getAnswerText());
    if (empty($answerText)) {
        // ❌ Пустой ответ = 0 баллов
        $answer->setAiScore(0);
        $answer->setAiComment('Ответ не был дан (время истекло или вопрос пропущен)');
        $answer->setProcessingStatus('completed');
        return;
    }
    
    // ... дальнейшая оценка для непустых ответов
}
```

## 🎯 Зачем отправлять пустые ответы?

### ✅ Преимущества:

1. **Полная история прохождения теста**
   - HR видит сколько вопросов было пропущено
   - Можно анализировать паттерны (какие вопросы сложные)

2. **Корректный подсчёт статистики**
   - Общее количество вопросов: 10
   - Ответов дано: 8 (из них 6 правильных, 2 неправильных)
   - Пропущено: 2

3. **Прозрачность для HR**
   - HR видит что сотрудник начал тест, но не ответил на часть вопросов
   - Можно понять: не знал ответ, не хватило времени, технические проблемы?

4. **Единообразие данных**
   - Каждый вопрос → всегда есть запись в `regulation_test_answer`
   - Проще делать JOIN запросы и аналитику

5. **Защита от эксплуатации**
   - Без пустых ответов: сотрудник мог бы умышленно не отвечать на сложные вопросы
   - С пустыми ответами: каждый пропуск = 0 баллов → честная оценка

  - 0 сек: время вышло → автоотправка

## 🐛 Исправленная проблема

### ❌ Было (до исправления):
```typescript
else {
  // Нет записи вообще - пропускаем вопрос
  console.log('No recording, skipping question...');
  skipToNextQuestion(); // ❌ Просто переходим дальше
}
```

**Проблема:**
- Вопрос пропускался без отправки на сервер
- В БД не было записи → HR не знал что вопрос был показан
- Статистика некорректная
- Таймер доходил до 0, выводился следующий вопрос, но ничего не отправлялось

### ✅ Стало (после исправления):
```typescript
else {
  // Нет записи вообще - отправляем пустой ответ
  console.log('No recording, submitting empty answer...');
  handleSubmitEmptyAnswer(); // ✅ Отправляем на сервер
}
```

**Результат:**
- ✅ Пустой ответ отправляется на сервер
- ✅ Создаётся запись в БД с `ai_score: 0`
- ✅ HR видит полную картину прохождения теста
- ✅ Статистика корректная

## 🚀 Тестирование

### Как проверить:

1. **Создать тест с коротким временем на вопрос:**
   ```
   maxTimePerQuestion = 10 секунд (для быстроты)
   ```

2. **Сценарий 1 - пропустить вопрос:**
   - Открыть тест
   - НЕ нажимать "🎤 Записать ответ"
   - Дождаться истечения таймера (10 сек)
   - **Ожидается:** 
     - Автоматический переход к следующему вопросу
     - В консоли: `"No recording, submitting empty answer..."`
     - В БД: запись с `answer_text = ''`, `ai_score = 0`

3. **Сценарий 2 - записать, но не отправить:**
   - Нажать "🎤 Записать ответ"
   - Сказать что-то
   - Нажать "⏹️ Остановить запись"
   - НЕ нажимать "✓ Отправить ответ"
   - Дождаться истечения таймера
   - **Ожидается:**
     - Автоматическая отправка готового blob
     - В консоли: `"Blob already exists, submitting..."`
     - В БД: запись с видео/аудио файлом + транскрипция + AI оценка

4. **Сценарий 3 - не успеть остановить запись:**
   - Нажать "🎤 Записать ответ"
   - Говорить всё время пока идёт таймер
   - Дождаться истечения таймера (не останавливать вручную)
   - **Ожидается:**
     - Автоматическая остановка записи
     - Автоматическая отправка после создания blob
     - В консоли: `"Recording in progress, stopping..."` → `"Blob created after auto-stop, submitting..."`
     - В БД: запись с видео/аудио файлом + транскрипция + AI оценка
