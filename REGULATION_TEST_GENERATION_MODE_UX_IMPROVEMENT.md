## 🎯 Почему это лучше

### 1. **Логическая группировка**
- ✅ Настройки генерации находятся на шаге генерации
- ✅ Шаг 1 теперь содержит только базовые настройки теста
- ✅ Меньше когнитивной нагрузки на первом шаге

### 2. **Контекстуальность**
- ✅ Пользователь выбирает режим генерации прямо перед генерацией
- ✅ Видит результат своего выбора сразу
- ✅ Может изменить режим перед генерацией

### 3. **Упрощение первого шага**
- ✅ Меньше полей для заполнения
- ✅ Быстрее начать создание теста
- ✅ Не нужно думать о генерации на начальном этапе

### 4. **Улучшенный flow**
```
Шаг 1: Что?     → Название и описание теста
Шаг 2: Откуда?  → Выбор регламентов
Шаг 3: Как?     → Режим генерации + генерация вопросов
Шаг 4: Кому?    → Приглашения
```

## 🧪 Тестирование

### Сценарий 1: Создание теста с предварительной генерацией

1. Перейти на `/hr/regulation-tests/create`
2. **Шаг 1:** Заполнить название и описание
   - ✅ Нет выбора режима генерации
3. Кликнуть "Далее"
4. **Шаг 2:** Выбрать регламенты
5. Кликнуть "Далее"
6. **Шаг 3:** 
   - ✅ Выбрать режим "Заранее"
   - ✅ Кликнуть "Сгенерировать вопросы"
   - ✅ Дождаться генерации

### Сценарий 2: Создание теста с динамической генерацией

1. Перейти на `/hr/regulation-tests/create`
2. **Шаг 1:** Заполнить название и описание
3. Кликнуть "Далее"
4. **Шаг 2:** Выбрать регламенты
5. Кликнуть "Далее"
6. **Шаг 3:** 
   - ✅ Выбрать режим "При старте"
   - ✅ Увидеть информационное сообщение
   - ✅ Кнопка "Сгенерировать" не отображается

### Сценарий 3: Изменение режима на шаге 3

1. Создать тест
2. Дойти до шага 3
3. Выбрать режим "Заранее"
4. Передумать и выбрать "При старте"
5. ✅ UI должен обновиться
6. Вернуться к "Заранее"
7. ✅ Появится кнопка генерации
