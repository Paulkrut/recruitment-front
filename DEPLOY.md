# 🚀 Деплой на Timeweb Cloud

## 📋 Требования

- Репозиторий в Bitbucket
- Dockerfile в корне репозитория
- Next.js приложение с настроенным `output: 'standalone'`
- **Node.js 22** (LTS версия)

## 🔧 Настройка проекта

### 1. Обновленный next.config.ts
```typescript
const nextConfig: NextConfig = {
  output: 'standalone', // Важно для Docker деплоя
  // ... остальные настройки
};
```

### 2. Dockerfile
Dockerfile настроен для production сборки с **Node.js 22**:
- Многоэтапная сборка для оптимизации размера
- Alpine Linux для минимального размера образа
- Безопасность: непривилегированный пользователь
- Порт 3000 для Timeweb Cloud

### 3. .dockerignore
Файл оптимизирует сборку, исключая ненужные файлы.

## 🌐 Деплой на Timeweb Cloud

### Шаг 1: Создание приложения
1. Войдите в панель Timeweb Cloud
2. Перейдите в раздел "Apps"
3. Нажмите "Создать приложение"

### Шаг 2: Выбор типа
- Выберите вкладку **"Dockerfile"**
- Это позволит использовать ваш собственный Dockerfile

### Шаг 3: Подключение репозитория
1. Выберите Bitbucket как VCS-провайдер
2. Укажите репозиторий: `your-username/recruitment-front`
3. Выберите ветку (обычно `main` или `master`)

### Шаг 4: Настройка переменных окружения
Добавьте необходимые переменные:
```
NODE_ENV=production
NEXT_PUBLIC_API_URL=https://your-backend-domain.com
```

### Шаг 5: Запуск деплоя
1. Нажмите "Создать приложение"
2. Дождитесь завершения сборки
3. Приложение будет доступно по указанному домену

## 🔍 Проверка работы

После успешного деплоя:
- Приложение будет доступно по домену Timeweb Cloud
- Nginx автоматически настроится на порт 3000
- Логи можно просматривать в панели управления

## 📝 Важные моменты

- **Порт**: Приложение слушает порт 3000 (указан в EXPOSE)
- **Сборка**: Используется многоэтапная сборка для оптимизации размера
- **Безопасность**: Приложение запускается от непривилегированного пользователя
- **Производительность**: Используется Alpine Linux для минимального размера образа

## 🚨 Возможные проблемы

### Ошибка сборки
- Проверьте, что все зависимости указаны в package.json
- Убедитесь, что yarn.lock актуален

### Ошибка запуска
- Проверьте логи в панели Timeweb Cloud
- Убедитесь, что порт 3000 правильно настроен

### Проблемы с переменными окружения
- Проверьте, что все необходимые переменные добавлены в настройках приложения
- Убедитесь, что переменные доступны во время сборки и запуска

## 🔄 Обновление приложения

При каждом push в основную ветку репозитория:
1. Timeweb Cloud автоматически обнаружит изменения
2. Запустится новая сборка
3. Приложение будет обновлено без простоя
