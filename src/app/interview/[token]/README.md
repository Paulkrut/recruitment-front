# ✅ Реализация показа информации о вакансии перед интервью

## 🎯 Задача
Добавить экран с информацией о вакансии для кандидатов, которые переходят по прямой ссылке, без дублирования для самозаписи.

## 📋 Что сделано

### 1. **Создан новый компонент `VacancyInfoStep.tsx`**
- ✅ Красивая карточка с информацией о вакансии
- ✅ Отображение данных кандидата
- ✅ Поддержка i18n (мультиязычность)
- ✅ Responsive дизайн
- ✅ Кнопка "Перейти к интервью"

**Отображает:**
- Название вакансии
- Компания
- Локация и зарплата
- Описание
- Обязанности
- Требования
- Имя и email кандидата

### 2. **Модифицирован `page.tsx` (страница интервью)**

**Минимальные изменения:**
- Добавлен импорт `useSearchParams`
- Добавлен импорт `VacancyInfoStep`
- Добавлено 3 новых `useState`:
  - `showVacancyInfo` - показывать ли экран
  - `vacancyData` - данные вакансии
  - `candidateData` - данные кандидата
- Изменена логика `activeStep` для учёта нового шага
- Обновлен `useEffect` для prepare - извлекает данные вакансии/кандидата
- Добавлено условие в `return` для показа `VacancyInfoStep`

**Не затронуто:**
- Вся логика интервью
- Проверка оборудования
- Запись ответов
- UI интервью
- > 99% существующего кода

### 3. **Модифицирован `apply/[token]/page.tsx` (самозапись)**
- При успешной записи добавляется `?skipVacancyInfo=true` к ссылке
- Кандидат после самозаписи НЕ увидит экран дважды

## 🔄 Flow кандидатов

### **Самозапись:**
```
/interview/apply/TOKEN 
→ Описание вакансии + форма
→ Заполняет форму
→ Редирект: /interview/HASH?skipVacancyInfo=true
→ Сразу проверка оборудования ✅
```

### **Прямая ссылка (новая):**
```
/interview/HASH
→ VacancyInfoStep (новый экран) ✅
→ Кнопка "Перейти к интервью"
→ Проверка оборудования
```

### **Прямая ссылка (старые ссылки):**
```
/interview/HASH (без параметра)
→ VacancyInfoStep (новый экран) ✅
→ Показывает инфо, если данные есть
→ Если данных нет - показывает что есть
```

## 🎨 Архитектурные решения

### ✅ **Модульность**
- Новый функционал в отдельном компоненте
- Минимум изменений в page.tsx (< 1%)
- Легко тестировать отдельно

### ✅ **Обратная совместимость**
- Старые ссылки продолжают работать
- Если нет данных вакансии - пропускает экран
- Не ломает существующий flow

### ✅ **Query параметр вместо БД**
- Нет миграций
- Нет новых таблиц
- Нет дополнительных запросов
- Простота = надёжность

### ✅ **Гибкость**
- Легко добавить новые поля к вакансии
- Легко изменить дизайн экрана
- Легко добавить аналитику

## 🔧 Backend requirements

API `/api/public/interview/{token}/prepare` должен возвращать:

```json
{
  "total": 10,
  "durationSec": 600,
  "status": "ready",
  "vacancy": {
    "title": "Frontend Developer",
    "company": "SofiHR",
    "description": "...",
    "location": "Москва",
    "salary": "150 000 - 200 000 ₽",
    "requirements": ["React", "TypeScript"],
    "responsibilities": ["Разработка UI", "Code review"]
  },
  "candidate": {
    "firstName": "Иван",
    "lastName": "Петров",
    "email": "ivan@example.com"
  }
}
```

**Опциональные поля:** `vacancy`, `candidate`, `location`, `salary`, `requirements`, `responsibilities`

Если `vacancy` или `candidate` отсутствуют - экран пропускается.
