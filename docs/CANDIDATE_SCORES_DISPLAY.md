# Отображение оценок кандидата

## Обзор

На странице кандидата теперь отображаются **две отдельные оценки** в едином визуальном блоке:
1. **Общая оценка по интервью** (знания, навыки, опыт)
2. **Оценка по компетенциям** (soft skills, личностные качества)

## Расположение

### 1. В шапке кандидата (компактно)
**Где:** В строке с Email, Телефон, Вопросов и т.д.  
**Формат:** Компактный, inline, с цветовой индикацией

```
Email: user@mail.ru • Телефон: +7... • Вопросов: 10 • 
🎓 Hard Skills: 8.2/10 • 💡 Fit: 7.5/10 • Создано: ...
```

### 2. На вкладке "Результаты" (детально)
**Где:** В самом верху вкладки, перед блоком "Детали интервью-сессии"  
**Формат:** Большая красивая карточка с градиентом и визуализацией

## Визуальное оформление

### Дизайн карточки:
- 🎨 **Светло-серый фон** (grey.50) - в стиле остальных блоков на странице
- 📊 **Две колонки** через Grid (адаптивные: вертикально на мобильных, горизонтально на десктопе)
- 🔄 **Вертикальный разделитель** между колонками на десктопе
- 📈 **Прогресс-бары** для визуализации оценок (height: 10px, borderRadius: 5px)
- 🏷️ **Цветовые чипы** с текстовой оценкой (Отлично/Хорошо/Удовлетворительно)
- 🎯 **Иконки** (SchoolIcon и PsychologyIcon) в primary.main цвете

### Структура блока:

```
┌───────────────────────────────────────────────────────────────┐
│  📊 Сводная оценка кандидата                                  │
├───────────────────────────────┬───────────────────────────────┤
│  🎓 Общая оценка по интервью  │  💡 Оценка по компетенциям   │
│     Оценка ответов на вопросы │     Soft skills и качества    │
│                               │                               │
│     8.2/10  [Хорошо]          │     7.5/10  [Хорошо]         │
│     ████████████████░░░░      │     ███████████████░░░       │
│     На основе 10 вопросов     │     6 компетенций: мотив...  │
└───────────────────────────────┴───────────────────────────────┘

[Светло-серый фон (grey.50), как у блока "Сводная информация"]
```

## Компонент

### Файл: `src/components/hr/CandidateScoresCard.tsx`

```typescript
interface CandidateScoresCardProps {
  interviewScore?: number;    // Оценка по вопросам (totalScore)
  competencyScore?: number;   // Средняя оценка по компетенциям
  questionsCount?: number;    // Количество вопросов
}
```

### Использование:

```tsx
<CandidateScoresCard
  interviewScore={sessionDetail?.result?.totalScore}
  competencyScore={competencyScore}
  questionsCount={sessionDetail?.answers?.length}
/>
```

## Две оценки: в чём разница?

### 1. Общая оценка по интервью (8.2/10)

**Что оценивается:**
- ✅ Правильность ответов на вопросы
- ✅ Профессиональные знания
- ✅ Технические навыки
- ✅ Опыт работы
- ✅ Hard skills

**Как формируется:**
- AI оценивает каждый ответ отдельно (0-10 баллов)
- Среднее арифметическое всех оценок
- Источник: `sessionDetail.result.totalScore`

**Когда используется:**
- Для оценки профессиональной компетентности
- "Знает ли кандидат свою работу?"
- Основа для технических позиций

---

### 2. Оценка по компетенциям (7.5/10)

**Что оценивается:**
- ✅ Мотивация к вакансии
- ✅ Культура речи
- ✅ Клиентоориентированность
- ✅ Стрессоустойчивость
- ✅ Ответственность
- ✅ Системное мышление
- ✅ Soft skills

**Как формируется:**
- AI анализирует все ответы целиком
- Оценивает 6 компетенций (0-10 каждая)
- Среднее арифметическое по компетенциям
- Источник: `evalData.metrics.summary_table.average_score`

**Когда используется:**
- Для оценки личностных качеств
- "Подходит ли кандидат по характеру и поведению?"
- Важно для клиентских и командных позиций

## Цветовая индикация

### Прогресс-бар и чипы:

| Оценка | Цвет | Текст | Значение |
|--------|------|-------|----------|
| 9-10 | 🟢 Зелёный | Отлично | Исключительный кандидат |
| 8-8.9 | 🟢 Зелёный | Хорошо | Сильный кандидат |
| 6-7.9 | 🟡 Жёлтый | Удовлетворительно | Приемлемый уровень |
| 4-5.9 | 🟡 Жёлтый | Ниже среднего | Требует внимания |
| 0-3.9 | 🔴 Красный | Плохо | Не рекомендуется |

## Обработка отсутствующих данных

### Если нет оценки по интервью:
```
🎓 Общая оценка по интервью
   Оценка ответов на вопросы
   
   Нет данных
```

### Если нет оценки по компетенциям:
```
💡 Оценка по компетенциям
   Soft skills и личностные качества
   
   Анализ компетенций не проведён
```

### Если нет обеих оценок:
Карточка всё равно отображается, но показывает "Нет данных" в обеих секциях.

## Интеграция на странице

### Код интеграции:

```tsx
// Вычисляем оценку по компетенциям
const competencyScore = aiMetrics && typeof aiMetrics === 'object' && 'summary_table' in aiMetrics 
  ? (aiMetrics as NewMetrics).summary_table?.average_score 
  : undefined;

// Вставляем компонент
<CandidateScoresCard
  interviewScore={sessionDetail?.result?.totalScore}
  competencyScore={competencyScore}
  questionsCount={sessionDetail?.answers?.length}
/>
```

## Преимущества такого размещения

### ✅ Для HR:
1. **Мгновенная оценка** - две ключевые метрики сразу видны
2. **Визуальное сравнение** - легко увидеть дисбаланс (хорошие знания, но плохие soft skills)
3. **Принятие решений** - понятно, нужен ли кандидат на техническую или клиентскую роль
4. **Приоритизация** - быстро отфильтровать слабых кандидатов

### ✅ Для UX:
1. **Выделенность** - оценки не теряются среди других данных
2. **Привлекательность** - красивый градиентный блок привлекает внимание
3. **Понятность** - чёткое разделение на две разные оценки
4. **Информативность** - не только цифры, но и прогресс-бары и текстовые пояснения

## Примеры сценариев

### Сценарий 1: Отличный профессионал, слабый fit
```
🎓 Hard Skills: 9.2/10 [Отлично]  ← Сильные знания
💡 Fit: 5.5/10 [Ниже среднего] ← Проблемы с мотивацией/коммуникацией
```
**Решение:** Подходит для технической роли без прямого контакта с клиентами

### Сценарий 2: Средние знания, отличный fit
```
🎓 Hard Skills: 6.5/10 [Удовлетворительно] ← Базовые знания
💡 Fit: 9.0/10 [Отлично] ← Отличное соответствие вакансии
```
**Решение:** Подходит для клиентских ролей, можно обучить технически

### Сценарий 3: Сбалансированный кандидат
```
🎓 Hard Skills: 8.0/10 [Хорошо]
💡 Fit: 8.2/10 [Хорошо]
```
**Решение:** Универсальный кандидат, подходит для любых ролей
