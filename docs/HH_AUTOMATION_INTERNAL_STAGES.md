# ✅ ЛОГИКА ПЕРЕДЕЛАНА - Внутренние стадии вместо HH стадий

## 🔄 ЧТО ИЗМЕНИЛОСЬ

### Было (на основе HH стадий):
```
Загружать кандидатов из HH стадии: [Отклик ▼]
После приглашения переводить в: [Приглашение ▼]
```

**Проблема:** Дублирование с настройками импорта, неясная логика

---

### Стало (на основе внутренних стадий): ✅
```
Отправлять приглашение кандидатам на стадиях:
  ☑️ 📥 Новые
  ☐ 🤖 AI Скрининг
  ☐ 📞 На связи
  ...
  
После приглашения → на стадию: [📞 На связи ▼]
Синхронизировать в HH стадию: [Приглашение ▼]
```

**Преимущества:**
- ✅ Логика основана на ваших внутренних стадиях
- ✅ Мультивыбор стадий (можно выбрать несколько)
- ✅ Нет дублирования с импортом
- ✅ По умолчанию выбрано "Новые"
- ✅ Четко понятно что происходит

---

## 📊 КАК ЭТО РАБОТАЕТ ТЕПЕРЬ

### Сценарий 1: Автоприглашение новых
```
1. Кандидат импортирован из HH
2. Попадает на стадию "📥 Новые"
3. ✅ Автоматически отправляется приглашение через HH.ru
4. Кандидат переводится на "📞 На связи"
5. В HH статус обновляется на "Приглашение"
```

---

### Сценарий 2: Приглашение после скрининга
```
Настройка:
  Отправлять приглашение на стадиях: ☑️ Финалист
  
Процесс:
1. HR смотрит резюме, проверяет AI скрининг
2. HR переводит кандидата на "⭐ Финалист"
3. ✅ Автоматически отправляется приглашение
4. Кандидат переводится на "📝 Тестирование"
```

---

### Сценарий 3: Мультистадии
```
Настройка:
  Отправлять приглашение на стадиях:
    ☑️ 📥 Новые
    ☑️ 🤖 AI Скрининг
    ☑️ ⭐ Финалист
  
Результат:
  Приглашения отправляются кандидатам на ЛЮБОЙ из этих стадий
```

## 🎨 UI ИЗМЕНЕНИЯ

### 1. Мультивыбор стадий
- Чекбоксы для множественного выбора
- Чипы показывают выбранные стадии
- По умолчанию: "📥 Новые"

### 2. Два поля вместо одного
- **"После приглашения → на стадию"** - куда переводить в вашей системе
- **"Синхронизировать в HH стадию"** - что обновить в HH.ru

### 3. Подсказки
```
💡 Приглашения отправляются через HH.ru только кандидатам с этих стадий
💬 Все приглашения отправляются через HH.ru
```

## 🔧 BACKEND - ЧТО НУЖНО ОБНОВИТЬ

### 1. Обновить логику в `HhAutomationService`:

**Было:**
```php
// Проверяем HH стадию
if ($candidate->getHhStageId() === $settings['fromHhStageId'])
```

**Стало:**
```php
// Проверяем внутреннюю стадию
$fromStages = $settings['fromInternalStages'] ?? [];
if (in_array($candidate->getStatus(), $fromStages))
```

### 2. После отправки приглашения:

```php
// Обновляем внутреннюю стадию
$candidate->setStatus($settings['toInternalStage']);

// Синхронизируем в HH
$this->hhApiService->updateCandidateStatus(
    $candidate->getHhCandidateId(),
    $settings['toHhStageId']
);
```
