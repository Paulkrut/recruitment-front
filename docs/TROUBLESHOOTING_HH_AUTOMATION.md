# 🔍 ДИАГНОСТИКА - Настройки автоматизации не отображаются

## ✅ ИСПРАВЛЕНО (2 проблемы)

### Проблема 1: Компонент не отображался
**Причина:** Возвращал `null` если настроек еще не было в базе.  
**Решение:** Добавлена инициализация дефолтных настроек.

### Проблема 2: Cannot find module 'unknown'
**Причина:** `@lingui/loader` пытался загрузить .po файлы, но модуль не настроен.  
**Решение:** Изменена загрузка на скомпилированные `.js` файлы.

## 🐛 ЕСЛИ ВСЁ ЕЩЁ НЕ ВИДНО

### Проверка 1: HH подключен?

В начале страницы должно быть:
```
✅ Интеграция с HeadHunter
[Зелёный чип: Подключено]
```

Если нет - компонент **не будет отображаться** (условие: `status?.isConnected && status.hasValidToken`).

---

### Проверка 2: DevTools Console

1. Нажмите `F12`
2. Откройте вкладку **Console**
3. Обновите страницу
4. Найдите ошибки (красные строки)

**Ожидаемое поведение:**
- Не должно быть ошибок импорта
- Не должно быть 404 на `/api/hh-integration/settings`

---

### Проверка 3: Network Tab

1. `F12` → вкладка **Network**
2. Обновите страницу
3. Найдите запрос: `GET /api/hh-integration/settings`

**Проверьте ответ:**
```json
{
  "success": true,
  "settings": null  // ← или {} или {...}
}
```

Если **404** - backend проблема, нужно проверить роуты.
