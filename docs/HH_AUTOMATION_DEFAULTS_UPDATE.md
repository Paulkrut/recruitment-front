# ✅ НАСТРОЙКИ ПО УМОЛЧАНИЮ ОБНОВЛЕНЫ

## 🎯 ЧТО ИЗМЕНИЛОСЬ

### 1. HH Стадия по умолчанию → "Тестовое задание" (assessment)

**Было:**
```typescript
toHhStageId: 'invitation' // Приглашение
```

**Стало:**
```typescript
toHhStageId: 'assessment' // Тестовое задание (fallback: phone_interview)
```

**Логика:**
1. Проверяем есть ли стадия `assessment` в списке HH стадий
2. Если нет - проверяем `phone_interview`
3. Если и её нет - берём первую доступную стадию
4. Автоматически обновляется при загрузке стадий из API

**Было:**

## 🔧 КАК РАБОТАЕТ FALLBACK ДЛЯ HH СТАДИЙ

### useEffect проверка:

```typescript
useEffect(() => {
  if (HH_STAGES.length > 0) {
    const hasAssessment = HH_STAGES.some(stage => stage.id === 'assessment');
    const hasPhoneInterview = HH_STAGES.some(stage => stage.id === 'phone_interview');
    
    setLocalSettings((prev: any) => {
      const currentStageId = prev.defaults.autoInvite.toHhStageId;
      const stageExists = HH_STAGES.some(stage => stage.id === currentStageId);
      
      // Если текущая стадия существует - оставляем
      if (stageExists) return prev;
      
      // Иначе: assessment → phone_interview → первая доступная
      const newStageId = hasAssessment ? 'assessment' 
                       : hasPhoneInterview ? 'phone_interview' 
                       : HH_STAGES[0]?.id || 'invitation';
      
      newSettings.defaults.autoInvite.toHhStageId = newStageId;
      return newSettings;
    });
  }
}, [HH_STAGES]);
```

### Сценарии:

**Сценарий 1:** У компании есть стадия "Тестовое задание"
```
✅ Устанавливается: assessment
```

**Сценарий 2:** Нет "assessment", но есть "phone_interview"
```
✅ Устанавливается: phone_interview
```

**Сценарий 3:** Нет обеих, берём первую доступную
```
✅ Устанавливается: HH_STAGES[0] (например, "invitation")
```

**Сценарий 4:** Настройки уже сохранены с существующей стадией
```
✅ Не меняем, оставляем сохранённую
```

1. ✅ **Сразу готово к работе** - оптимальные настройки по умолчанию

## 🔄 МИГРАЦИЯ СУЩЕСТВУЮЩИХ НАСТРОЕК

Если у компании уже есть сохранённые настройки:
- **Не меняются** - остаются как есть
- Fallback срабатывает только если стадия не существует в HH

Если компания настраивает первый раз:
- **AI по умолчанию** ✅
- **assessment (или phone_interview)** ✅
- **Новые кандидаты** ✅
- **Переводить в "На связи"** ✅
